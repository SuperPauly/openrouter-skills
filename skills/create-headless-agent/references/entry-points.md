# Entry Points

Alternative entry points for the headless agent. The CLI entry point is generated by default.

## Contents

- [CLI](#cli) (DEFAULT ON) — args/stdin to agent to stdout
- [HTTP Server](#http-server) — Bun.serve() with SSE streaming
- [MCP Server](#mcp-server) — expose agent as MCP tool

---

## CLI

The primary entry point. Accepts a prompt via `--prompt`, positional argument, or piped stdin. Supports three output modes: text (default), JSON event stream, and quiet (exit code only).

### src/cli.ts

The `#!/usr/bin/env bun` shebang on line 1 is required so the OS can execute the file directly when it's invoked via the `bin` symlink created by `bun link`. Without it, the linked binary fails with "Exec format error".

```python
import argparse

def parse_cli_args() -> argparse.Namespace:
    parser = argparse.ArgumentParser()
    parser.add_argument("--model", default="openrouter/auto")
    parser.add_argument("--session-id")
    return parser.parse_args()
```

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Agent or API error |
| 2 | Output validation error (schema mismatch) |

### Usage

```bash
# From --prompt flag
bun run src/cli.ts --prompt "List all Python files"

# From positional argument
bun run src/cli.ts "What is 2+2?"

# From piped stdin
echo "Summarize this file" | bun run src/cli.ts

# JSON event stream (NDJSON, one AgentEvent per line)
bun run src/cli.ts --json --prompt "Search for TODO comments"

# Quiet mode for CI/scripts (exit code only)
bun run src/cli.ts --quiet --prompt "Fix the linting errors" && echo "Success"

# Override model
bun run src/cli.ts -m anthropic/claude-sonnet-4 -p "Review this code"

# Limit cost and steps
bun run src/cli.ts --max-cost 0.50 --max-steps 10 -p "Refactor utils.ts"

# Validate structured output
bun run src/cli.ts --output-schema schema.json -p "Extract entities as JSON"

# Disable session persistence for a one-off run
bun run src/cli.ts --no-session -p "Quick question"
```

### package.json scripts

```json
{
  "scripts": {
    "start": "bun run src/cli.ts",
    "dev": "bun --watch src/cli.ts"
  }
}
```

## HTTP Server

A `Bun.serve()` HTTP server that exposes the agent over a REST API with optional SSE streaming. Not generated by default -- add this when the agent needs to be accessed over HTTP.

### src/server.ts

```python
from http.server import BaseHTTPRequestHandler, HTTPServer
import json

class HealthHandler(BaseHTTPRequestHandler):
    def do_GET(self) -> None:
        body = json.dumps({"ok": True}).encode()
        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.end_headers()
        self.wfile.write(body)
```

### Usage

```bash
# Start the server
bun run src/server.ts

# With auth and CORS configured
AGENT_API_SECRET=my-secret ALLOWED_ORIGIN=https://myapp.com bun run src/server.ts
```

```bash
# Health check
curl http://localhost:3000/health

# Non-streaming request
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Hello"}'

# Streaming (SSE)
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Hello", "stream": true}'

# With auth
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer my-secret" \
  -d '{"prompt": "Hello"}'

# With message history
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Hi"}, {"role": "assistant", "content": "Hello!"}, {"role": "user", "content": "What did I say?"}]}'
```

## MCP Server

Expose the agent as an MCP tool so other agents (including Claude Code) can call it. Uses `@modelcontextprotocol/sdk` with the stdio transport. Not generated by default -- add this when the agent should be composable with other MCP-aware tools.

### src/mcp-server.ts

```python
from dataclasses import dataclass

@dataclass
class RunAgentRequest:
    prompt: str
    model: str | None = None

def run_agent_tool(request: RunAgentRequest) -> dict:
    selected_model = request.model or "openrouter/auto"
    return {"model": selected_model, "output": f"Processed: {request.prompt}"}
```

### Dependencies

```bash
bun add @modelcontextprotocol/sdk
```

### Claude Code MCP configuration

Add to `.claude/settings.json` or `~/.claude.json`:

```json
{
  "mcpServers": {
    "my-agent": {
      "command": "bun",
      "args": ["run", "src/mcp-server.ts"],
      "cwd": "/path/to/my-agent"
    }
  }
}
```

Once configured, Claude Code can call the agent via the `run_agent` tool:

```
Use the my-agent tool to search for TODO comments in the codebase.
```
