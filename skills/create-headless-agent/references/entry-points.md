## CLI

The primary entry point. Accepts a prompt via `--prompt`, positional argument, or piped stdin. Supports three output modes: text (default), JSON event stream, and quiet (exit code only).

### src/cli.ts

The `#!/usr/bin/env bun` shebang on line 1 is required so the OS can execute the file directly when it's invoked via the `bin` symlink created by `bun link`. Without it, the linked binary fails with "Exec format error".

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Agent or API error |
| 2 | Output validation error (schema mismatch) |

### Usage

```bash
# From --prompt flag
bun run src/cli.ts --prompt "List all Python files"

# From positional argument
bun run src/cli.ts "What is 2+2?"

# From piped stdin
echo "Summarize this file" | bun run src/cli.ts

# JSON event stream (NDJSON, one AgentEvent per line)
bun run src/cli.ts --json --prompt "Search for TODO comments"

# Quiet mode for CI/scripts (exit code only)
bun run src/cli.ts --quiet --prompt "Fix the linting errors" && echo "Success"

# Override model
bun run src/cli.ts -m anthropic/claude-sonnet-4 -p "Review this code"

# Limit cost and steps
bun run src/cli.ts --max-cost 0.50 --max-steps 10 -p "Refactor utils.ts"

# Validate structured output
bun run src/cli.ts --output-schema schema.json -p "Extract entities as JSON"

# Disable session persistence for a one-off run
bun run src/cli.ts --no-session -p "Quick question"
```

  }

```bash
# Health check
curl http://localhost:3000/health

# Non-streaming request
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Hello"}'

# Streaming (SSE)
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Hello", "stream": true}'

# With auth
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer my-secret" \
  -d '{"prompt": "Hello"}'

# With message history
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Hi"}, {"role": "assistant", "content": "Hello!"}, {"role": "user", "content": "What did I say?"}]}'
```

### Claude Code MCP configuration

Add to `.claude/settings.json` or `~/.claude.json`:

```json
{
  "mcpServers": {
    "my-agent": {
      "command": "bun",
      "args": ["run", "src/mcp-server.ts"],
      "cwd": "/path/to/my-agent"
    }
  }
}
```
